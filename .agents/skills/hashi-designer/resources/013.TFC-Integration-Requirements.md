# TFC Integration Requirements

UI changes needed in HCP Terraform Cloud to support invisible agent experience.

> **Design Direction**: See [UXDR-014](./014.UXDR-TFC-Integration.md) for UX principles.
> The TFC UI is for **configuration and exception handling**, not monitoring.
> GitHub remains the primary interface for scan results.

---

## 2a. Deep Link Parameters (P0)

TFC must handle these query parameters on workspace/project settings pages to support the OAuth return flow and external linking.

| Parameter | Values | Behavior |
|-----------|--------|----------|
| `iac_scanning` | `enabled`, `disabled` | Set toggle state and show confirmation |
| `status` | `success`, `error`, `pending` | Display appropriate banner |
| `error_code` | `auth_denied`, `org_required`, `token_failed`, `webhook_failed` | Show specific error message |
| `auth_redirect` | `true` | Trigger GitHub OAuth flow immediately |

**Example URLs:**

```
# Success return after OAuth
/app/{org}/workspaces/{ws}/settings/integrations?iac_scanning=enabled&status=success

# Error return - org approval needed
/app/{org}/workspaces/{ws}/settings/integrations?status=error&error_code=org_required

# Direct link to trigger OAuth
/app/{org}/workspaces/{ws}/settings/integrations?auth_redirect=true
```

**Banner Messages:**

| Status | Banner |
|--------|--------|
| `success` | ✅ IaC Risk Scanning enabled successfully. Your first scan will run within the hour. |
| `error` + `auth_denied` | ❌ Authorization cancelled. Toggle scanning on to try again. |
| `error` + `org_required` | ⚠️ Organization approval required. Ask an org admin to approve the GitHub App. |
| `error` + `token_failed` | ❌ Connection failed. Please try again. [Retry] |
| `pending` | ⏳ Waiting for GitHub authorization... |

## 2b. Notification Preferences (P0)

**Location**: Project Settings > Integrations > IaC Risk Scanning > Notifications

Users can only "set and forget" if they trust they'll be notified when action is needed. This is critical for the invisible experience.

```

│                                                                         │

## 3. Workspace Settings: Post-Activation View

After scanning is enabled, show connection status. The default view is minimal—expanded details available on demand.

### 3a. Clean State (Most Common)

When everything is working and no risks are open, show minimal UI:

```
┌─────────────────────────────────────────────────────────────────────────┐
│  IaC Risk Scanning                                      [● ON  ○ OFF]   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │

│                                                                         │

| Status | Display | When |
|--------|---------|------|

**Requirements:**
| Requirement | Description |
|-------------|-------------|
| Real-time updates | Poll or SSE, update column in place |
| Three states only | Running / Success / Failed |
| Relative timestamps | "2m ago", "1h ago", "Just now" |
| Risk count inline | Show severity breakdown in Risks column |
| No agent names | Implementation detail, hidden by default |
| Retry on failure | Failed state shows inline retry action |
| Bulk actions in overflow | "Scan All Now" in menu, not prominent |
